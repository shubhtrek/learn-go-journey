# Step 2.5: Error Handling (No Try-Catch) ⚠️

Go handles errors by returning them as normal values.

## Key Concepts
1. **Errors as values**: Explicitly checked via `if err != nil`.
2. **`errors.New`**: Basic way to create a new error message.
3. **`fmt.Errorf`**: Creates an error using format strings.

## How to Run
```bash
go run main.go
```
